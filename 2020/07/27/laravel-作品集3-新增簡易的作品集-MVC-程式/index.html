[laravel][作品集3] 新增簡易的作品集 MVC 架構

規劃設計一個能有左選單樹進行分類的作品集介面，如下圖:

設計MVC簡易架構

新增作品集的 MVC 三支程式:
  • Model : portfolioCategory.php
  • View : portfolioIndex.blade.php
  • Controller : portfolioController.php

網址請求 http://127.0.0.1:8000/portfolio 的時候路由程式 routes/ web.php 會導向 controller後，controller 程式再使用 index 方法初始並顯示使用 Model 從資料庫撈取的資料於作品集的頁面。

Controller路由導向View

於resources / views 目錄下手動新增 作品集的 .blade.php頁面 portfolioIndex.blade.php ![圖:頁面 portfolioIndex.blade.php](/images/pasted-10.png)

於 portfolioIndex.blade.php 新增測試文字，方便後續測試。

下指令新增 controller:

php artisan make:controller portfolioController

在 app / Http / Controllers 目錄下會有一支新增的 portfolioController.php 程式

在 Controller 中新增 index() 方法，並在 路由程式 web.php 導向到index()方法時，
return view 到 頁面 portfolioIndex.blade.php

在程式 routes/ web.php 新增http get 導向的語法，導向到 portfolioController.php 程式的 index()方法。

Route::get('/portfolio','portfolioController@index');

http://127.0.0.1:8000/portfolio 網址可成功顯示頁面:

從Controller撈取Model資料並顯示於View

因為下一篇文章將會以樹選單開始撰寫，因此這篇文章將以新增作品分類的資料模組為練習範例。
一樣下 make 的指令，新增一個作品分類的 Model

php artisan make:model portfolioCategory

下完指令後，會在 app的根目錄新增出 Model

新增 protected $table 參數，指定之前篇章中新增於資料庫的 作品分類資料表:

對應作品分類表:

新增好作品分類的資料模型後，可使用簡易的方法來測試呈現Model資料是否正常。
先在 Controller 中引用(use) app根目錄下的該作品分類Model，

use App\portfolioCategory;

使用該Model 的靜態方法 ::all() 將所有資料取出:

$portfolioCategory = portfolioCategory::all();

使用 view 將變數另外存於 response 之中，並傳給 View

return view("portfolioIndex", [ 'portfolioCategory' => $portfolioCategory ]);
